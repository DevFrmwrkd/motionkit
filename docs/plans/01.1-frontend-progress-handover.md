## 1. High-Level Architecture & Layout

The app uses **Next.js 16.2** + **React 19** with **Tailwind CSS v4** and **Shadcn UI**. 
The central workspace is located at `app/src/app/workstation/page.tsx` and features a 3-panel production UI. We have also completed the user-facing Marketplace, Settings, Dashboard, and Creator Studio pages.

## 2. Completed Frontend Routes & Components

### 2.1 Landing Page (`app/src/app/page.tsx`)
- **Hero & Value Prop**: Beautiful gradients, strong CTAs ("Try Workstation", "Browse Presets").
- **Feature Grid**: Showcasing BYOK, schema-driven forms, encrypted keys, and real-time preview.

### 2.2 Auth Pages (`app/src/app/(auth)/...`)
- **Login & Signup (`/login`, `/signup`)**: Clean, minimalist authentication forms. Fully responsive with social login buttons and standard email/password inputs.

- **Dashboard Layout**: Fixed left sidebar for navigation, top header, and scrollable content area.

- **Upload Wizard (`/creator/upload`)**: A two-step process:

  - Step 2: Details form (Name, Description, Price, Category) and a preview of the "Parsed Schema Fields" from the bundle.

### 2.9 Workstation (`app/src/app/workstation/page.tsx`)
- **Top Bar**: Global navigation, plus "Save Preset" and "Add to Project" interactive modal dialogs (using Shadcn `Dialog`).

- **Right Panel (`InputControls.tsx`)**: The dynamic property controls (`SchemaForm.tsx`) and the main Render button.

### 2.10 Core Video & Schema Engine
- **`app/src/remotion/presets/HelloWorld/index.tsx`**: We built a fully working "Hello World" preset conforming to our `PresetExport` contract (`Component`, `schema`, `meta`). It uses `useCurrentFrame`, `interpolate`, and `spring` for smooth animations.
- **`PresetPlayer.tsx`**: A wrapper around `@remotion/player` that takes `component`, `inputProps`, and `meta`.
- **`SchemaForm.tsx`**: Dynamically creates UI controls based on the preset's `PresetSchema` (supports text, color, slider, etc., using `react-colorful`).

## 3. What the Next Agent Needs to Do (Backend & Convex)

The frontend is waiting for real data and backend mutations. 

### 3.1 Convex Setup
- A Convex environment is already set up in `convex/schema.ts` (users, collections, savedPresets, renderJobs, projects).
- **Task**: The next agent needs to implement the Convex mutations (e.g., `convex/mutations.ts` or `convex/render.ts`) to create render jobs.

### 3.2 Wire up the Workstation Page
In `app/src/app/workstation/page.tsx`, replace the frontend simulation with real hooks:
1. **Render Button**: Replace the simulated `handleRender` with a call to `useMutation(api.renders.createJob)`.
   - Pass `presetId` (e.g., 'hello-world'), `inputProps` (JSON.stringify), and `bundleUrl` (or internal ID).
2. **Render Queue**: Replace the hardcoded `renderJobs={[]}` array with `useQuery(api.renders.listJobs)`. This will feed live render statuses into the `RenderQueue.tsx` component.
3. **Preset Library**: Replace the hardcoded `[{ id: "hello-world"... }]` with a query to fetch the available presets from Convex (e.g. `useQuery(api.presets.list)`).

### 3.3 Wire up the Dashboards & Marketplace
1. **Dashboard Overview/Projects/Collections**: Fetch data from `api.projects.list`, `api.collections.list`, and `api.renders.listJobs`.
2. **Creator Studio**: Wire up the file upload zone in `/creator/upload` to store the `.zip` in Convex/Cloudflare R2, parse the module, and save it to the `presets` table.
3. **Marketplace Listing**: Fetch presets from Convex via `useQuery(api.presets.list)`.
4. **Settings**: Hook up the API key forms to save securely into Convex `users` table via `useMutation(api.users.updateKeys)`.

### 3.4 The Rendering Engine
- **Task**: The backend agent must set up the **Modal** or **Lambda** serverless functions to actually execute `@remotion/cli` rendering when a Convex action is triggered.
